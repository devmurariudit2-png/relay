## 2024-05-09 - [O(n^2) nested loops in ReconcileService.js and reconcileEngine.js]
**Learning:** The reconciliation logic nested two large arrays for "Pass 2" (finding exact amount + date within 3 days). A 20k array length resulted in ~8 seconds execution time because of O(N * M) behavior checking all internal vs all bank transactions. Memory mentions avoiding O(n^2) and pre-grouping for performance optimization without sacrificing correctness.
**Action:** Replace the nested loop by indexing the internal transactions by rounded amount first (O(M)), and then for each bank transaction, only loop over the `internal` transactions that have the same exact amount (O(N * candidates)). This maintains the same functional constraints (exact amount + 3 day diff window) but speeds up the operation by over 100x.
