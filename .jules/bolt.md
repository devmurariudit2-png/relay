## 2024-05-20 - [O(N^2) Reconciliation Pattern]
**Learning:** The transaction reconciliation logic (e.g., in `ReconcileService.js` and `reconcileEngine.js`) processes large datasets and is highly performance-critical. The Pass 2 matching used a nested loop leading to O(N^2) complexity. When converting to an O(N) Map lookup, exact matching on amounts fails due to floating point inaccuracies and the 0.01 tolerance crossing rounding boundaries.
**Action:** When matching amounts within a threshold (like < 0.01), pre-group candidates in a Map using a rounded value (e.g., `Math.round(amount * 100)`), and then evaluate lookups checking offsets `key - 1`, `key`, and `key + 1` to prevent missing valid matches.
## 2024-05-20 - [CI E2E and Unit Tests Fixes]
**Learning:** CI was failing because Playwright tests were nested inside the Relay-main directory causing Vitest backend test runner collisions. Also, backend test jobs required missing native rollup bindings. In addition, the Playwright E2E job requires explicit `@playwright/test` installation, and `VITE_SUPABASE_URL`, `SUPABASE_URL` dummy environment variables setup correctly.
**Action:** Relocated `*.spec.js` Playwright E2E tests to an `e2e` directory at the root level, explicitly removed legacy nested `playwright.config.js` files, and created a single unified `playwright.config.js` at the root. Updated `.github/workflows/ci.yml` to supply necessary environment variables and native bindings.
